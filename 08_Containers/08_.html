To access JupyterLab or another web app running within the container, it is necessary to publish ports from the container to the host system. To understand this, some background in networking is needed. An IP address is the identifier of a computer in a network. But when one wants to share data between two entities within a network, we want it to go exactly where it is needed, that is why we need ports. Ports allow the system to specify the origin and the destination of data. Since the container is isolated from the computer, they can't share information. Publishing a port binds the port in the container to another one and thus allows everybody in the network to access the service, unless a firewall forbids it. You can get more information in this video https://www.youtube.com/watch?v=g2fT-g9PX9o

Beside Continuous Integration, containers may make your life much easier. Whether a compiler is not available in your OS, or you don't want to waste time installing another version of it, containers can save you a lot of time. VS Code Dev Containers brings the ease to another level, as only a couple of clicks are enough to set everything.

To transfer data between the container and the host system, you can either mount data or copy it. To work with python 2.7 for example, the first method can be done with the following command:

docker run --rm --name MyPython2 -v ${pwd}:/home/Tom -it python:2.7 /bin/bash

-v ${pwd}:/home/Tom mounts the current directory in the host system to the user folder Tom in the container. --rm sets automatic deletion for the container once it is stopped. -it lets us interact with the container. /bin/bash is the command we use to start a bash shell from within the container.
