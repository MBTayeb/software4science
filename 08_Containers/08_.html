To access JupyterLab or another web app running within the container, it is necessary to publish ports from the container to the host system. To understand this, some background in networking is needed. An IP address is the identifier of a computer in a network. But when one wants to share data between two entities within a network, we want it to go exactly where it is needed, that is why we need ports. Ports allow the system to specify the origin and the destination of data. Since the container is isolated from the computer, they can't share information. Publishing a port binds the port in the container to another one and thus allows everybody in the network to access the service, unless a firewall forbids it. You can get more information in this video https://www.youtube.com/watch?v=g2fT-g9PX9o

Beside Continuous Integration, containers may make your life much easier. Whether a compiler is not available in your OS, or you don't want to waste time installing another version of it, containers can save you a lot of time. VS Code Dev Containers brings the ease to another level, as only a couple of clicks are enough to set everything.

To transfer data between the container and the host system, you can either mount data or copy it. To work with python 2.7 for example, the first option is done while starting the container. The following command mounts the current directory to the user folder Tom in the container:

docker run --rm --name MyPython2 -v ${pwd}:/home/Tom -it python:2.7 /bin/bash

--rm sets automatic deletion for the container once it is stopped.
--name sets a name for the container we are starting.
-v ${pwd}:/home/Tom mounts the current working directory in the host system to the user folder Tom in the container.
-it lets us interact with the container.
/bin/bash is the command we use to start a bash shell from within the container.

Note: Instead of a path in the host system, the container can also be mounted to a volume, which is a directory that is easily accessible for containers. It allows for sharing and persisting data between containers, providing flexibility and scalability in managing data within a containerized environment. (Read more in https://docs.docker.com/storage/volumes/)

The second method, that is copying data between the container and the host system, is done when the container is already running. Analogous to the first option, let us start the container and then copy the current directory to the user folder Tom in the container:

docker run --rm --name MyPython2 -d -it python:2.7 /bin/bash
docker cp ${pwd} MyPython2:/home/Tom
docker exec -it MyPython2 /bin/bash

We use -d while starting the container to run it in the background. The last line is needed to access and interact with the running container.
