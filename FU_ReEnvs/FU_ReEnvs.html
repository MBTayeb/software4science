How to set up reproducible python environments:

 1) Set up your environments per project: Stick with dedicated virtual environments for each project. Global library installations are convenient at first, but cumbersome in the long run.

 2) Keep track of used python version, libraries and their exact versions: For Python several ways exist to achieve this (see below for details on the available tools). A straightforward way is to list Python version and libraries and their versions in text files in your projects, often named "runtime.txt" (Python version) and "requirements.txt" (libraries). (`environment.yml` from conda etc. is an alternative.) Instead of these files one can also use pipenv with a pipfile, which additionally 'freezes' the environment (and therefore protects it against changes). 

Available tools:

  • pypi.org/pip, see e.g. https://fangohr.github.io/introduction-to-python-for-computational-science-and-engineering/18-environments.html

  •  Binder/repo2docker, see e.g.  https://arxiv.org/abs/2102.09562 and https://www.desy.de/~fangohr/publications/posters/2023-05-11_fangohr_JupyterCon_reproducible-workflows-with-jupyter.pdf
