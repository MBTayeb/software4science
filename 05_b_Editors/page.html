CLI Editors:

How you edit your code is largely a question of taste, but some tools like Vi are so widely used (and available on basically every remote server you might want to access) that you should have an idea of their basic usage, and some tools provide certain functions (for building, linking, debugging, testing, managing code) that you don't want to miss anymore once you are used to them.

Nano

Nano is a CLI editor that is available by default in a lot of Linux distros (distributions) and can also be installed if needed. The cool thing about it is that it gets the job done, that is, it's very beginner friendly.

https://www.youtube.com/watch?v=NV9PyPJKqH4

Vi-m

Like mentioned above, Vi is also widely used. It has a relatively long learning curve, but can make the editing experience much faster. In 1991, an improved clone of Vi got released. It is called Vim, that is, VI IMproved. It introduced syntax highlighting and many other things. Now, Vi can't be installed, it comes by default in UNIX systems. The latest Linux distros don't ship vi anymore and often use vi as an alias for vim. Vim can be installed though. To customize Vim, one writes a Vimscript. One can basically custom anything in the editor. Then, a fork of vim got released in 2015. It has the name Neovim (abbreviated as nvim). This fork aims to be hyperextensible with Lua (another programming language) and is still compatible with Vimscript.

Common integrated development environments(IDEs):

Microsoft IDEs

Microsoft offers two popular IDEs, Visual studio Code (VS Code) and Visual Studio. Their names are similar but they are quite different. VS Code is one of the most used IDEs in the world. It is available on all operation systems and can be used with all programming languages. Visual Studio, on the other hand, is mainly used for developing .Net or C++ under the Windows OS.
