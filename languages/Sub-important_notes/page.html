Doing RSEng work usually means to write (source) code in a programming language, using editors or integrated development environments (IDEs) to do so efficiently. The proper choice of these elements (languages, editors or IDEs) is not obvious or unambiguous, so it is worth the time to spend some thoughts on this issue.

There is a very large number of programming languages available and although for most of them there are good reasons why they are used for their specific applications, some languages a certainly better suited for scientific work. A core difference between languages is whether they must be compiled (turned into machine-readable 'binaries' via a compiler before execution) or whether they are 'interpreted', i.e. written as scripts which are 'read' and executed by a separate program. Fortran would be an example for the former, Python for the latter.

Compiled languages usually have static typing (variables must be defined as being of a certain type, e.g. a real number, or a string of characters), but compilation can be optimized in several ways and is hardware-specific, so that compiled languages are considered to deliver high performance. Accordingly, most 'high performance computing' (HPC) applications are based on compiled languages like Fortran or C/C++. Interpreted languages usually have dynamic typing (the type of variables is determined upon execution, depending on what values are given) and are considered to be better suited for the quick and easy development of software solutions, also since there is no time-consuming compilation step involved. Some new languages like Julia promise to provide both high performance and easy development with 'just-in-time' compilation. (Although for now I don't think Julia can be recommended as a first programming language, you will probably hear a lot of praise for Julia from experienced HPC people, which is why I have added some information on Julia as a separate work unit below.) For more information, watch this video https://www.youtube.com/watch?v=I1f45REi3k4 .

Apart from performance, the existing software 'ecosystem' of a language is very important for the proper choice of a programming language. Fortran, 'the mother tongue of scientific computing' (C.E. Leith), is still at the center of HPC, first of all because there is a huge ecosystem of existing code, libraries, tools, etc. available, that makes it easy to develop scientific applications in fortran, especially concerning so-called 'number-crunching' tasks (the computation of huge numbers of mathematical operations). C/C++ is arguably the most widely used language for 'high-level' commercial work, so that a large pool of professional software tools is available for C/C++, which has made C/C++ very attractive for larger scientific software projects in recent years. Considering interpreted or scripting languages, Python is well established in many fields in- and outside of science, so that with this programming language easy access to new technologies is available (cloud computing, machine learning etc.).

Once you have mastered one or two programming languages, you will see that the core elements of programming languages are rather universal and that learning new languages is often not more complicated than memorizing a new syntax and probably keeping a few 'odd' features in mind. (The concept of 'object oriented programming', or OOP, is an exception: it is the basis for OOP-languages like C++ and Python and we will therefore have to learn what it is about in the section devoted to Python below.) This means that when choosing your programming languages you don't have to worry too much about ending up in a dead end. Considering performance and environment, the choice of programming languages for scientific applications is practically often less confusing than it initially seems, as it often simply makes sense to stick to 'popular' solutions, for which a large pool of clever people constantly work on both performance and new tools.

You will find that while Java, C/C++ and Python are most popular for professional software development, the most popular languages in science are Python, C/C++, Fortran (for HPC), and 'less general' languages like R, Matlab, etc. These lists often do not include scripting languages like Bash, which underlie the functionality of command line interfaces themselves (CLIs, which basically means the terminals you're typing into), which is why in addition everyone should have at least a basic understanding of 'their' CLI language.
