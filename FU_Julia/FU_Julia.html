Some people think Julia is the next big thing in HPC. I'm personally a little bit skeptical, but that should not prevent us from taking a look at it:

Julia is in development since 2009 and was first released to the public in 2012; it is thus a rather new, but free and open source language. The idea is to combine high performance with easy usage, especially for HPC applications. It is a compiled language and thus nearly as fast as C, but can be written like a scripting language thanks to 'just-in-time' compilation and optional typing. (Types are run-time objects and, depending on the actual value of variables, fitting binary-versions are chosen at run-time, which is a feature called 'multiple dispatch'.) C and Fortran functions can be called directly from Julia, Python functions via PyCall, so that the vast existing body of code in C, Fortran and Python can be smoothly integrated. Many Julia basics have the look and feel of Fortran.

Unlike Python, Julia has built-in vectors and matrices available. Control structures and functions are delimited with end statements, code indentation is recommended but serves no specific function. Julia is not object-oriented, although one could argue that one can of course program object-oriented with any language, but functions (methods) are not 'owned' by classes in Julia.

Will Julia be the successor of Fortran for HPC applications? We don't know yet, but there is certainly a chance that this could happen. The current 'ecosystem' of available tools is still rather small, but constantly growing. BTW: The 'Ju' in Jupyter is for Julia, which is indeed fully supported by Jupyter notebooks.
