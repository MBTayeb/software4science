This section provides a short overview of additonal often more advanced topics in the field of RSEng.

Code Review and Documentation:
The proper maintenance of larger software projects requires regular code review (programmers are going together through existing code) and documentation. Documentation can range from source code comments over a proper manual (maybe online and/or generated via a tool like Doxygen) up to 'literate programming', where the actual code is embedded in a text on what for, why and how a code works.

Research Compendia:
Code is also data and in this sense proper research data management must also consider managing, sharing and archiving code. A research compendium attempts to do this in the form of a collection of software, data, etc. besides ’the paper’ where 'of course everything' can be found.
For more information, see https://research-compendium.science/

Shared Environments and Reproducibility:
Reproducibility in science is a topic for research software engineering. Proper code development and management is necessary to guarantee reproducibility. Shared environments or environments for sharing code like Jupyter Notebooks and Hubs or Binder are an excellent option to increase reproducibility by allowing to share code that can in addition be executed within defined environments easily. You will find a separate work unit on Jupyter Notebooks below.

Software & Data Citation & Licensing:
At some point the citation and licensing of software can become an issue for researchers. One structural problem of current science is that very often scientific research software engineering is not covered by the commonly used 'success metrics' for scientific careers. Proper citation of software should thus be of high importance to code developers. Licensing is important to keep in mind when starting to share or otherwise publish code. There is a separate focus unit on this topic available below.
