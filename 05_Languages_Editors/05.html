Keeping all this in mind I would like to recommend the following strategy here: Start with learning Bash, so that you feel comfortable operating a Linux system especially when writing scripts to automate your work. Then visit Fortran; it's still very much at the center of scientific computing and the basics are not hard to get; it is very straightforward mathematical programming. For doing your actual research move on to Python: it is much less demanding than C++ and usually everything you could only wish for not to write yourself will be available. Once your project turns quasi-professional or if you anyhow have the intention to make it that way, C++ will offer everything you need (but be prepared to spend most of the time planning to write code, instead of actually writing it). Finally, learn R, Matlab, SQL etc. if you have to, because a project requires it - you certainly won't find it impossible to do so once you have worked through this course.

In this course, we will follow the strategy outlined above and focus on Bash, Fortran and especially Python. (This strategy might not fit to your scientific field, especially if the whole field -- like certain parts of bio- or geo-informatics -- developed long after the high times of Fortran; in this case start with bash and move on to R, Matlab or whatever is the accepted standard in your field.) If you have worked through this unit, you have learned some 'universal' features of programming languages already:
First, variables need to be assigned (and sometimes declared, i.e. given a type).
Second, the flow of programs requires control structures like loops, conditions etc.
And third, sub-units (functions, sub-routines, modules, etc.) are needed to structure code.
Apart from this, there is some general stuff to remember for each language, like e.g. what line defines the start of a program or how do I do in- and output etc. When learning a new programming language, you are oftentimes fine with remembering just the basics. If you then look for the correct 'syntax' of an element, it's usually best practice to google it ("fortran loop"), once you have understood what you are looking for.

Online resources for learning Fortran:
Fortran is a compiled language, so that apart from Fortran itself you will have to learn how to use a compiler including the use of software libaries and as compiling can be quite tedious for all but very small projects, also about the most common 'build system' make.
One of the reasons why Fortran provides such excellence performance is the existence of very high-quality compilers (the Intel Fortran compiler is a legend), which manage to produce extremely fast code. But unlike in the past, C++ compilers (including the GCC one) don't seem to be utterly behind anymore.
The use of software libraries is essential for good programming! Libraries provide code written by someone else (usually more clever than you or me) that can be re-used for your purposes just by sticking to the calling conventions (you have to supply the correct inputs and understand what is given back). If you learn how to use libraries you can save a lot of time by not re-rewriting existing code, which in addition is usually much more error-proof than the ugly hack you just made up.
The use of a build system like make is also considered good programming, as it allows not only you, but also other people to compile your code conveniently (or: at all).
You will learn more about compilers, software libraries and build systems in the next work unit.
