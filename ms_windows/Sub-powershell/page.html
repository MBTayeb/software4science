PowerShell

Windows PowerShell is a built-in command line shell in the Windows OS. Just like the other Windows command line shell "cmd.exe", also known as "Command Prompt", it is a scripting language, which means that it can be used to automate tasks. What differentiates it from cmd is the fact that it is a configuration management framework (it can be used to automatically track and control many computers at once). Thus, it can do everything a cmd can do and much more. This is why it has been named PowerShell. cmd.exe scripts are called "batch files" and have the extensions ".cmd" or ".bat", while PowerShell scripts have the extension ".ps1".

PowerShell can be compared to the Linux Bash shell. However, it is systematically different. The output of Bash commands is in text format, while the output of PowerShell commands is in object format (.NET objects). This can cause a lot of confusion for those who are used to Bash. Moreover, all the commands in PowerShell are called cmdlets (pronounced command-lets) and named in the form of "verb-noun", which could be helpful for beginners. This is very different from bash though, where the commands are usually one word. For example, "Get-ChildItem" is the equivalent of "ls" in bash (short for list). Still, there are many built-in aliases in PowerShell that rename the commands to their Bash equivalent (e.g. "ls" works also on PowerShell).

Starting from the 6th version, PowerShell has become drastically different from the previous versions. So the newer ones are called PowerShell Core to differentiate them from the old ones, i.e Windows PowerShell. The reason behind this is that the older ones, like PowerShell 5, are built on the .Net Framework while the new ones are built on .Net Core. This change was done to make PowerShell available on macOS and Linux too. However, there are still some cmdlets that are not available on macOS and Linux. Furthermore, there are are cmdlets that are only available in macOs and Linux. For more information, visit https://learn.microsoft.com/en-us/powershell/scripting/whats-new/unix-support . Another major difference is that PowerShell core is open-source.
