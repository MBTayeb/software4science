Linux Environment within a Windows Computer: WSL/MinGW

As mentioned before, Windows is often not the OS many programming languages are built for. Now we will discuss the ways one can solve this problem.

WSL

Windows Subsystem for Linux is a Microsoft service that allows you to run a Linux OS within Windows. This has been a revolutionary step from Windows, as it allows one to have the best of both Worlds. This has improved further with the WSL 2, with which one can open many GUIs. There are many WSL distributions available directly in the Microsoft Store. We recommend using Ubuntu, as it is the most popular Linux distribution and has the most support.

The following tutorials may get a bit advanced for beginners, the first parts of each tutorial are usually enough for most users. More advanced users can continue with the rest.

MinGW

MinGW (Minimalist GNU for Windows) is a tool that allows you to run many Linux packages natively on Windows. For example, one could use the GNU compilers like gcc for C and g++ for C++. This is a good alternative for those who don't want to use WSL. However, it is limited as one can only use the packages that are offered by MinGW. The initial MinGW project only offers a 32-bit compiler, but the binaries can be executed in both 32 and 64 bit. This is not interesting, as it is not updated anymore.

In 2007, a fork was published which compiles into 64-bit, which is why it is called MinGW-w64. To use this, there are various possibilities. Strawberry Perl is an environment for the Perl interpreter on Windows, which includes MinGW-w64. This method is maybe the most straightforward. However, you can only use the binaries included. For more customization, MSYS2 offers an environment very similar to Linux on Windows. It was built on Cygwin, which is another alternative that we don't need to introduce here. After installing MSYS2, different environment are available. The MSYS environment is the parent environment, so the others inherit from it. These environment differ by the compilers used for building their packages and other things. The package-manager used in MSYS2 is pacman, which is the same as in Arch Linux.
