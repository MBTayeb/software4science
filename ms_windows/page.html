Since more than 70% of all computers worldwide run on Microsoft Windows, we will discuss here some relevant things for those using it as their operating system. Unfortunately, many programming tools are designed for Unix-based systems, which can cause a lot of trouble and waste a lot of time. We will list here some tools that can make your life much easier as a Windows user. We recommend using only supported versions of Windows, because many tools are not available for older versions. You can check whether your version is supported in the Windows release health page https://learn.microsoft.com/en-us/windows/release-health/ .

Package Managers

Whether one is used to the Linux Packet managers or not, package managers can save much time. There are three main ones available on Windows, i.e. Winget, Chocolatey and Scoop. Winget is made by Microsoft, very simple to use and doesn't normally require an installation for Windows 10 and later versions. Chocolatey is made by the community and has more packages available. However, it must first be installed and needs an elevated shell for each package installation. We recommend using Winget, since it should be enough for most users.

Windows Terminal and prompt customization

Windows Terminal is a must-have as a Windows user. It is a terminal emulator and an improvement of the old Windows Console. It brings the CLI experience to another level.

In addition to that, Oh-My-Posh can make your experience even more enjoyable. It customizes the prompt you see in the terminal very easily. You can pick one of many themes. Since it changes the prompt, each shell (PowerShell, cmd.exe, Bash ...) used within the computer needs a separate setting. Typically, people add it to the shell profile, which is a file used to configure what should be loaded in the start of a new shell session. If one wants to use it within a WSL, one would need to install it in WSL.

PowerShell

Windows PowerShell is a built-in command line shell in the Windows OS. Just like the other Windows command line shell "cmd.exe", also known as "Command Prompt", it is a scripting language, which means that it can be used to automate tasks. What differentiates it from cmd is the fact that it is a configuration management framework (it can be used to automatically track and control many computers at once). Thus, it can do everything a cmd can do and much more. This is why it has been named PowerShell. cmd.exe scripts are called "batch files" and have the extensions ".cmd" or ".bat", while PowerShell scripts have the extension ".ps1".

PowerShell can be compared to the Linux Bash shell. However, it is systematically different. The output of Bash commands is in text format, while the output of PowerShell commands is in object format (.NET objects). This can cause a lot of confusion for those who are used to Bash. Moreover, all the commands in PowerShell are called cmdlets (pronounced command-lets) and named in the form of "verb-noun", which could be helpful for beginners. This is very different from bash though, where the commands are usually one word. For example, "Get-ChildItem" is the equivalent of "ls" in bash (short for list). Still, there are many built-in aliases in PowerShell that rename the commands to their Bash equivalent (e.g. "ls" works also on PowerShell).

Starting from the 6th version, PowerShell has become drastically different from the previous versions. So the newer ones are called PowerShell Core to differentiate them from the old ones, i.e Windows PowerShell. The reason behind this is that the older ones, like PowerShell 5, are built on the .Net Framework while the new ones are built on .Net Core. This change was done to make PowerShell available on macOS and Linux too. However, there are still some cmdlets that are not available on macOS and Linux. Furthermore, there are are cmdlets that are only available in macOs and Linux. For more information, visit https://learn.microsoft.com/en-us/powershell/scripting/whats-new/unix-support . Another major difference is that PowerShell core is open-source.

Linux Environment within a Windows Computer: WSL/MinGW

As mentioned before, Windows is often not the OS many programming languages are built for. Now we will discuss the ways one can solve this problem.

WSL

Windows Subsystem for Linux is a Microsoft service that allows you to run a Linux OS within Windows. This has been a revolutionary step from Windows, as it allows one to have the best of both Worlds. This has improved further with the WSL 2, with which one can open many GUIs. There are many WSL distributions available directly in the Microsoft Store. We recommend using Ubuntu, as it is the most popular Linux distribution and has the most support.

The following tutorials may get a bit advanced for beginners, the first parts of each tutorial are usually enough for most users. More advanced users can continue with the rest.

MinGW

MinGW (Minimalist GNU for Windows) is a tool that allows you to run many Linux packages natively on Windows. For example, one could use the GNU compilers like gcc for C and g++ for C++. This is a good alternative for those who don't want to use WSL. However, it is limited as one can only use the packages that are offered by MinGW. The initial MinGW project only offers a 32-bit compiler, but the binaries can be executed in both 32 and 64 bit. This is not interesting, as it is not updated anymore.

In 2007, a fork was published which compiles into 64-bit, which is why it is called MinGW-w64. To use this, there are various possibilities. Strawberry Perl is an environment for the Perl interpreter on Windows, which includes MinGW-w64. This method is maybe the most straightforward. However, you can only use the binaries included. For more customization, MSYS2 offers an environment very similar to Linux on Windows. It was built on Cygwin, which is another alternative that we don't need to introduce here. After installing MSYS2, different environment are available. The MSYS environment is the parent environment, so the others inherit from it. These environment differ by the compilers used for building their packages and other things. The package-manager used in MSYS2 is pacman, which is the same as in Arch Linux.
