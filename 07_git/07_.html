Proper code management is very important even for medium-sized programming projects. Version control systems and source code repositories are essential elements for a proper code management strategy.

Version control systems:

There a a number of version control systems available (subversion, cvs, ...), but git seems to have taken over the field for most purposes. Git is an open-source, distributed version control system. The idea is that you 'check in' your local (maybe off-line) working copy of your code project into a central repository. This way, not only your code is save and changes can be tracked and rolled back if problems occur, but you can also work together with other people on a shared code. It requires the system to spot conflicting changes and this is indeed what git does: It simply accepts 'harmless' changes, but it raises your awareness if attempted changes would collide with the work of your colleagues.

After the setup of your code repository (git init), the system has to allow for code to be checked out (git checkout)and changes to be submitted, included or merged (git merge). The addition of changes happens stepwise by first adding files (if necessary) not only to your working directory, but also the git meta data on your working directory (git add) and by registering changes on your files as to be commited (git commit) and then 'push' your changes to the server (git push). You see that the basic git commands are quite straightforward, but there is of course a lot more functionality available (and needed for large projects with many contributors).
