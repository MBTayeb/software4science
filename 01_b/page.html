The target group of the course are students or researchers who need to write software for their project or research or those who just want to learn programming. These people come from different fields and don't normally have a structured software engineering training. This leads often to low quality software, as such people are using software only as a personal tool without sharing it with the readers. In other words, readers are not able to reproduce the software, which makes the verification of the results challenging. Thus, research itself suffers.

To face this challenge, the field of Research Software engineering (RSEng) was created. It is concerned with all technical, intellectual and also social issues concerning the development of software in scientific contexts. Important topics are programming languages, programming tools, etc. but also the challenges we face when 'translating' theoretical considerations with a scientific background (a 'scientific theory') into algorithms that can be implemented as source code for software, as well as issues like the reproducibility, sharing of code etc.

The motto here is Better Software, Better Research.
