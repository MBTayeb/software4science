Why Python? Because it is an open source software for all major operating systems with an extremely large ecosystem covering almost everything from web frameworks to scientific programming. Python is interpreted, not compiled and therefore usally not as fast as Fortran or C/C++ for number crunching applications, but it is fully object-oriented with dynamic typing. We already mentioned the Anaconda environment above, as well as the PyCharm IDE and Jupyter notebooks (guess what the 'pyt' stands for). BTW: If the question turns up; Python 2 is outdated, use Python 3.

Python basics are similar to most scripting languages, 'container' data types allow for lists etc., indexing starts like for C/C++ at zero. A 'def' statement is used to define functions, and they, as well as all other control structures, are delimited by identation - this means you have to do proper identation (four spaces), otherwise your functions and control structures won't work, but you don't need any end statements. Python is as mentioned above proper object-oriented, which means that everything in Python is an object (e.g., a=1 corresponds to an object a of class int with the value 1).

Object oriented programming (OOP):
OOP puts the idea of modularization to the max. It bundles data with methods into so-called objects, derived from classes. Three OOP concepts are at the core of this.

Encapsulation: Bundling of data with methods and restriction of outside access. This means that each object usally comes with fitting methods and data variables. Such object is an 'instance' of a class, with which the properties of objects of this kind are defined.
Inheritance: Basing classes or objects upon other classes or objects. This means that it is possible to pass properties from ’parent’ to ’child’ classes and therefore objects that are instances of these classes.
Polymorphism: Methods with same names can have different functionality in different (sub-)classes. This means it is possible to use methods in parent classes across sub-classes of different type. (This makes the setup of parent classes much easier ...)
