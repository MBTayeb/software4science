Once your projects has grown to more than a handful of source code files (to make your project accessible to anyone but yourself, each source code file should not contain more than a few hundred lines of code), the process of compiling your program can become very tedious, especially if you have different compilers, options, variables etc. for compilation on different systems. This is were build systems like make come into play; they allow you to specify rules for handling files and thereby to automate the process of compilation. Once you download a code that makes proper use of make, you just have to change into the source code directory and type make to do all necessary compilation steps. The makefile itself can be generated on the target system, using system specific variables and options. This is why most projects require you to do the 'classic' three step procedure to compile and install a new program:

  1. ./configure
  2. make
  3. make install

The first one generates the specific makefile, the second does the compilation and the third command copies the produced binaries to a pre-defined installation directory like $HOME/bin, /usr/bin, etc. Package management systems are built on top of this; they additionally resolve 'dependencies', i.e. if a program requires other software (usually libraries) to run, these can also be automatically installed.
